# Security Policy

## Supported versions

The Ghost Trees Bundle is currently in active development.
Security fixes are provided for the latest minor version of the
latest major version.

| Version | Supported          |
|---------|--------------------|
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a vulnerability

If you discover a security vulnerability in this bundle, please
**do not open a public issue**.

Instead, report it privately via GitHub's "Report a vulnerability"
feature on the [Security tab](https://github.com/ericgansa/ghost-trees-bundle/security)
of the repository, or by contacting the maintainer directly.

Your report should include:

- a description of the vulnerability,
- steps to reproduce,
- the affected version(s),
- the potential impact,
- any suggested mitigation if you have one.

You will receive an acknowledgment within **5 working days**, and
a status update within **15 working days**.

## Disclosure policy

Once a vulnerability is confirmed:

1. A fix is prepared in a private branch.
2. A new patch release is published.
3. A GitHub Security Advisory is published with credit to the reporter
   (unless they prefer to remain anonymous).

We follow [coordinated disclosure](https://en.wikipedia.org/wiki/Coordinated_disclosure):
public details about the vulnerability are not disclosed until a fix
is available.
